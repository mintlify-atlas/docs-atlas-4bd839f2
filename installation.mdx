# Installation Guide

This comprehensive guide covers installing EduStack Smart for both development and production environments. Follow these instructions to deploy a fully functional academic management platform.

<Info>
  For a quick development setup, see the [Quick Start Guide](/quickstart). This guide provides more detailed instructions for various deployment scenarios.
</Info>

## System Requirements

### Minimum Requirements

#### Server Requirements

- **Operating System**: Linux (Ubuntu 20.04+, Debian 11+, CentOS 8+) or macOS
- **Web Server**: Nginx 1.18+ or Apache 2.4+
- **PHP**: 8.2 or higher (8.3 recommended)
- **Database**: PostgreSQL 13+ or SQLite 3.35+
- **Memory**: 2GB RAM minimum, 4GB+ recommended
- **Storage**: 10GB minimum, SSD recommended
- **Node.js**: 18.x or higher
- **Process Manager**: Supervisor or systemd (for queue workers)

#### PHP Extensions

The following PHP extensions are required:

<CodeGroup>
```bash Ubuntu/Debian
sudo apt-get install -y \
  php8.3-cli \
  php8.3-fpm \
  php8.3-mbstring \
  php8.3-xml \

  php8.3-gd \
  php8.3-pgsql \
  php8.3-sqlite3 \
  php8.3-redis \
  php8.3-bcmath \
  php8.3-intl
```

```bash CentOS/RHEL
sudo yum install -y \
  php83-cli \
  php83-fpm \
  php83-mbstring \
  php83-xml \
  php83-curl \
  php83-zip \
  php83-gd \
```

#### Additional Software

- **Composer**: 2.5+ (PHP dependency manager)
- **pnpm**: 8.0+ (Node.js package manager, faster than npm)
- **Git**: 2.30+ (version control)
- **Redis**: 6.0+ (optional, for caching and sessions)

### Recommended Production Requirements

- **CPU**: 4+ cores
- **Memory**: 8GB+ RAM
- **Storage**: 50GB+ SSD
- **PostgreSQL**: 14+ with dedicated server
- **Redis**: Separate Redis instance for caching
- **CDN**: CloudFlare or AWS CloudFront for static assets
- **SSL Certificate**: Let's Encrypt or commercial SSL

<Warning>
  Do not use SQLite in production. PostgreSQL is strongly recommended for production deployments.
</Warning>

## Application Installation

### Step 1: Clone Repository

```bash
# Navigate to your web root or projects directory
cd /var/www  # or ~/projects for development

# Clone the repository
git clone <repository-url> edustack-smart
cd edustack-smart

# Set proper ownership (production)
sudo chown -R www-data:www-data /var/www/edustack-smart
```

### Step 2: Install Dependencies

<Steps>
  <Step title="Install PHP Dependencies">
    ```bash
    composer install --optimize-autoloader --no-dev
    ```
    
    <Note>
      For development, omit the `--no-dev` flag to include development dependencies like Pest and Laravel Pint.
    </Note>
    
    This installs:
    - **Laravel Framework 12**: Core framework
    - **Laravel Sanctum**: SPA authentication
    - **Laravel Reverb**: WebSocket server
    - **Inertia.js**: Server-side adapter
    - **Spatie MediaLibrary**: Media management
    - **Spatie Permission**: Role-based access control
    - **AWS SDK**: Cloud storage support
  </Step>
  
  <Step title="Install Node Dependencies">
    ```bash
    pnpm install --frozen-lockfile
    ```
    
    This installs:
    - **React 19**: UI library
    - **TypeScript**: Type safety
    - **Inertia.js React**: Frontend adapter
    - **Tailwind CSS 4**: Utility-first CSS
    - **Radix UI**: Accessible components
    - **Vite 7**: Build tool
    - **TipTap**: Rich text editor
    - **React Query**: Data fetching
  </Step>
</Steps>

### Step 3: Environment Configuration

<Steps>
  <Step title="Create Environment File">
    ```bash
    cp .env.example .env
    ```
  </Step>
  
  <Step title="Generate Application Key">
    ```bash
    php artisan key:generate
    ```
    
    This generates a secure encryption key for the application.
  </Step>
  
  <Step title="Configure Environment Variables">
    Edit `.env` with your production settings:
    
    ```bash .env
    # Application
    APP_NAME="EduStack Smart"
    APP_ENV=production
    APP_KEY=base64:YOUR_GENERATED_KEY
    APP_DEBUG=false
    APP_URL=https://your-domain.com
    
    # Locale
    APP_LOCALE=en
    APP_FALLBACK_LOCALE=en
    
    # Database
    DB_CONNECTION=pgsql
    DB_HOST=127.0.0.1
    DB_PORT=5432
    DB_DATABASE=edustack_smart
    DB_USERNAME=edustack_user
    DB_PASSWORD=your_secure_password
    
    # Redis Cache (recommended)
    CACHE_STORE=redis
    REDIS_CLIENT=phpredis
    REDIS_HOST=127.0.0.1
    REDIS_PASSWORD=null
    REDIS_PORT=6379
    
    # Session
    SESSION_DRIVER=redis
    SESSION_LIFETIME=120
    
    # Queue
    QUEUE_CONNECTION=redis
    
    # Broadcasting (WebSockets)
    BROADCAST_CONNECTION=reverb
    
    # Reverb Configuration
    REVERB_APP_ID=your_app_id
    REVERB_APP_KEY=your_app_key
    REVERB_APP_SECRET=your_app_secret
    REVERB_HOST=your-domain.com
    REVERB_PORT=443
    REVERB_SCHEME=https
    
    # File Storage
    FILESYSTEM_DISK=local
    MEDIA_DISK=public
    
    # AWS S3 (optional)
    AWS_ACCESS_KEY_ID=
    AWS_SECRET_ACCESS_KEY=
    AWS_DEFAULT_REGION=us-east-1
    AWS_BUCKET=
    
    # Mail
    MAIL_MAILER=smtp
    MAIL_HOST=smtp.mailtrap.io
    MAIL_PORT=2525
    MAIL_USERNAME=your_username
    MAIL_PASSWORD=your_password
    MAIL_FROM_ADDRESS="noreply@your-domain.com"
    MAIL_FROM_NAME="${APP_NAME}"
    
    # Logging
    LOG_CHANNEL=stack
    LOG_LEVEL=error
    ```
    
    <Warning>
      Always set `APP_DEBUG=false` in production to prevent sensitive information leakage.
    </Warning>
  </Step>
</Steps>

### Step 5: Database Setup

<Steps>
  <Step title="Run Migrations">
    ```bash
    php artisan migrate --force
    ```
    
    The `--force` flag is required in production. This creates:
    - Users table with two-factor authentication
    - Roles and permissions tables
    - Posts, categories, and types tables
    - Projects and collaborators tables
    - Events and activities tables
    - Media library tables
    - Cache, sessions, and jobs tables
  </Step>
  
  <Step title="Create Storage Link">
    ```bash
    php artisan storage:link
    ```
    
    This creates a symbolic link from `public/storage` to `storage/app/public`.
  </Step>
  
  <Step title="Seed Default Data (Optional)">
    ```bash
    php artisan db:seed --class=ProductionSeeder
    ```
    
    Seeds:
    - Default roles and permissions
    - Admin user account
    - Basic configuration
    
    <Info>
      For development with sample data, use `php artisan db:seed` without the class parameter.
    </Info>
  </Step>
</Steps>

### Nginx Configuration

<CodeGroup>
```nginx /etc/nginx/sites-available/edustack-smart
server {
    listen 80;
    listen [::]:80;
    server_name your-domain.com www.your-domain.com;
    
    # Redirect to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name your-domain.com www.your-domain.com;
    
    root /var/www/edustack-smart/public;
    index index.php index.html;
    
    # SSL Configuration
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    
    # Security Headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    
    # Logging
    access_log /var/log/nginx/edustack-access.log;
    error_log /var/log/nginx/edustack-error.log;
    
    # Gzip Compression
    gzip on;
    gzip_vary on;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/xml+rss application/json;
    
    # Main location
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    
    # PHP-FPM
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_hide_header X-Powered-By;
    }
    
    # Deny access to hidden files
    location ~ /\. {
        deny all;
    }
    
    # Static files caching
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## Queue Worker Setup

### Using Supervisor (Recommended)

<Steps>
  <Step title="Install Supervisor">
    ```bash
    sudo apt-get install -y supervisor
    ```
  </Step>
  
  <Step title="Create Configuration">
    Create `/etc/supervisor/conf.d/edustack-worker.conf`:
    
    ```ini
    [program:edustack-worker]
    process_name=%(program_name)s_%(process_num)02d
    command=php /var/www/edustack-smart/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
    autostart=true
    autorestart=true
    stopasgroup=true
    killasgroup=true
    user=www-data
    numprocs=4
    redirect_stderr=true
    stdout_logfile=/var/www/edustack-smart/storage/logs/worker.log
    stopwaitsecs=3600
    ```
  </Step>
  
  <Step title="Start Worker">
    ```bash
    sudo supervisorctl reread
    sudo supervisorctl update
    sudo supervisorctl start edustack-worker:*
    
    # Check status
    sudo supervisorctl status
    ```
  </Step>
</Steps>

### Using Systemd

<Accordion title="Alternative: systemd Service">
  Create `/etc/systemd/system/edustack-worker.service`:
  
  ```ini
  [Unit]
  Description=EduStack Smart Queue Worker
  After=network.target
  
  [Service]
  Type=simple
  User=www-data
  WorkingDirectory=/var/www/edustack-smart
  ExecStart=/usr/bin/php /var/www/edustack-smart/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
  Restart=always
  RestartSec=3
  
  [Install]
  WantedBy=multi-user.target
  ```
  
  Enable and start:
  ```bash
  sudo systemctl daemon-reload
  sudo systemctl enable edustack-worker
  sudo systemctl start edustack-worker
  sudo systemctl status edustack-worker
  ```
</Accordion>

## Laravel Reverb Setup (WebSockets)

<Steps>
  <Step title="Configure Reverb">
    Update `.env` with Reverb settings:
    
    ```bash .env
    BROADCAST_CONNECTION=reverb
    
    REVERB_APP_ID=edustack
    REVERB_APP_KEY=$(openssl rand -hex 16)
    REVERB_APP_SECRET=$(openssl rand -hex 32)
    REVERB_HOST=your-domain.com
    REVERB_PORT=8080
    REVERB_SCHEME=https
    ```
  </Step>
  
  <Step title="Create Supervisor Config">
    Create `/etc/supervisor/conf.d/edustack-reverb.conf`:
    
    ```ini
    [program:edustack-reverb]
    command=php /var/www/edustack-smart/artisan reverb:start
    autostart=true
    autorestart=true
    user=www-data
    redirect_stderr=true
    stdout_logfile=/var/www/edustack-smart/storage/logs/reverb.log
    ```
    
    Start the service:
    ```bash
    sudo supervisorctl reread
    sudo supervisorctl update
    sudo supervisorctl start edustack-reverb
    ```
  </Step>
  
  <Step title="Configure Nginx Proxy">
    Add to your Nginx server block:
    
    ```nginx
    # WebSocket proxy for Laravel Reverb
    location /app {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
    ```
    
    Reload Nginx:
    ```bash
    sudo nginx -t && sudo systemctl reload nginx
    ```
  </Step>
</Steps>

    ```bash
    tail -f /var/www/edustack-smart/storage/logs/laravel.log
    tail -f /var/log/nginx/edustack-error.log
    ```
    
    Common causes:
    - Permissions issues on `storage/` or `bootstrap/cache/`
    - Missing `.env` file or `APP_KEY`
    - Database connection issues
  </Accordion>
  
  <Accordion title="Queue Jobs Not Processing">
    Check worker status:
    ```bash
    sudo supervisorctl status edustack-worker:*
    ```
    
    View worker logs:
    ```bash
    tail -f /var/www/edustack-smart/storage/logs/worker.log
    ```
    
    Restart workers:
    ```bash
    php artisan queue:restart
    sudo supervisorctl restart edustack-worker:*
    ```
  </Accordion>
  
  <Accordion title="WebSocket Connection Failed">
    Check Reverb is running:
    ```bash
    sudo supervisorctl status edustack-reverb
    ```
    
    Test WebSocket connection:
    ```bash
    curl -i -N -H "Connection: Upgrade" -H "Upgrade: websocket" -H "Sec-WebSocket-Version: 13" -H "Sec-WebSocket-Key: test" http://localhost:8080/app/edustack
    ```
  </Accordion>
  
  <Accordion title="Permission Denied Errors">
    Reset permissions:
    ```bash
    sudo chown -R www-data:www-data /var/www/edustack-smart
    sudo chmod -R 755 /var/www/edustack-smart
    sudo chmod -R 775 /var/www/edustack-smart/storage
    sudo chmod -R 775 /var/www/edustack-smart/bootstrap/cache
    ```
  </Accordion>
</AccordionGroup>
