## Prerequisites

Before you begin, ensure you have the following installed:

- **PHP 8.2 or higher** - Laravel 12 requires PHP 8.2+
- **Composer** - PHP dependency manager
- **Node.js 18+** - For frontend asset compilation
- **pnpm** - Preferred package manager (or npm/yarn)
- **SQLite** or **MySQL/PostgreSQL** - Database engine
- **Git** - Version control

## Initial Setup

### 1. Clone the Repository

```bash
git clone <repository-url>
cd edustack-smart
```

### 2. Install Dependencies

Install PHP dependencies:

```bash
composer install
```

Install JavaScript dependencies:

```bash
pnpm install
```

### 3. Environment Configuration

Copy the example environment file:

```bash
cp .env.example .env
```

Generate an application key:

```bash
php artisan key:generate
```

### 4. Database Setup

For SQLite (default):

```bash
touch database/database.sqlite
php artisan migrate --seed
```

For MySQL/PostgreSQL, update your `.env` file with database credentials and run:

```bash
php artisan migrate --seed
```

## IDE Helper Generation

EduStack Smart includes `laravel-ide-helper` for better IDE autocomplete. Generate helper files:

```bash
php artisan ide-helper:generate
php artisan ide-helper:models
php artisan ide-helper:meta
```

These commands generate helper files that improve IDE autocomplete for facades, models, and meta information.

## Debugging Tools

### Laravel Pail

Laravel Pail is included for real-time log viewing. Start Pail to monitor application logs:

```bash
php artisan pail
```

Pail will display logs in real-time with color-coded output. Options:

```bash
# Filter by log level
php artisan pail --filter=error

# Filter by message content
php artisan pail --message="User logged in"

# Set timeout (0 for no timeout)
php artisan pail --timeout=0
```

Pail is automatically included in the `composer run dev:ssr` command.

### Laravel Tinker

Tinker provides an interactive REPL for Laravel:

```bash
php artisan tinker
```

Use Tinker to:
- Test Eloquent queries
- Interact with models
- Execute artisan commands
- Debug application logic

### Xdebug

#### Installation

Install Xdebug via PECL:

```bash
pecl install xdebug
```

#### Configuration

Add to your `php.ini`:

```ini
zend_extension=xdebug.so
xdebug.mode=debug
xdebug.start_with_request=yes
xdebug.client_port=9003
xdebug.client_host=127.0.0.1
```

#### VS Code Configuration

Install the **PHP Debug** extension and add to `.vscode/launch.json`:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Listen for Xdebug",
      "type": "php",
      "request": "launch",
      "port": 9003,
      "pathMappings": {
        "/var/www/html": "${workspaceFolder}"
      }
    }
  ]
}
```

## Code Quality Tools

### Laravel Pint

Laravel Pint is included for PHP code formatting:

```bash
# Format all files
vendor/bin/pint

# Format specific files
vendor/bin/pint app/Models

# Test without making changes
vendor/bin/pint --test
```

### ESLint & Prettier

For JavaScript/TypeScript code quality:

```bash
# Run ESLint
pnpm run lint

# Format code with Prettier
pnpm run format

# Check formatting without changes
pnpm run format:check
```

### TypeScript Type Checking

Check TypeScript types:

```bash
pnpm run types
```

## Database Tools

### TablePlus / DBnD

Use a database GUI client for easier database management:

**Connection Settings:**
- Host: `127.0.0.1`
- Database: `database/database.sqlite` (for SQLite) or your configured database name
- User: As configured in `.env`
- Password: As configured in `.env`

### Laravel Migrations

Manage database schema:

```bash
# Run migrations
php artisan migrate

# Rollback last migration
php artisan migrate:rollback

# Reset and re-run all migrations
php artisan migrate:fresh

# Reset and seed
php artisan migrate:fresh --seed
```

## Troubleshooting

### Common Issues

**Permission Errors**

If you encounter permission errors:

```bash
chmod -R 775 storage bootstrap/cache
```

**Node Module Issues**

If you have issues with node modules:

```bash
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

**Composer Issues**

Clear Composer cache:

```bash
composer clear-cache
composer install
```

**Port Already in Use**

If port 8000 is already in use:

```bash
php artisan serve --port=8001
```
