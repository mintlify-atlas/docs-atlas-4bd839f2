# Teacher Guide

Teachers in EduStack Smart can create and manage academic projects, invite student collaborators, and publish project work. This guide covers all teacher-specific features.

## Overview

As a teacher, you can:
- Create and edit projects
- Manage project collaborators
- Track project versions
- Publish or hide projects
- Edit project content with a visual editor
- Manage project screenshots and galleries

## Creating a New Project

### Starting a Project

<Steps>
  <Step title="Navigate to Projects">
    Click on "Proyectos" (Projects) in the main navigation menu.
  </Step>
  
  <Step title="Create New Project">
    Click the "Crear Proyecto" (Create Project) button.
  </Step>
  
  <Step title="Fill Project Details">
    Complete the project creation form with all required information (see below).
  </Step>
  
  <Step title="Upload Images">
    Add at least one screenshot or image of your project using the drag-and-drop zone.
  </Step>
  
  <Step title="Submit">
    Click "Crear Proyecto" (Create Project) to save your project.
  </Step>
</Steps>

### Project Information Fields

When creating a project, you'll need to provide:

#### Basic Information

- **Nombre** (Name): Project title (minimum 3 characters)
- **Resumen** (Summary): Project description (minimum 100 characters)
  - Use this space to explain what the project does
  - Describe the problem it solves
  - Mention key features

#### Technical Details

- **Repositorio URL**: Link to your code repository (GitHub, GitLab, etc.)
  - Must be a valid URL starting with http:// or https://
  - Example: `https://github.com/username/project`

- **URL del Proyecto**: Live demo or deployment URL
  - Link to where the project can be accessed
  - Example: `https://myproject.example.com`

- **Versión** (Version): Project version number
  - Use semantic versioning (e.g., 1.0.0, 2.1.3)
  - Default: 1.0.0

- **Licencia** (License): Software license
  - Common options: MIT, GPL, Apache 2.0
  - Default: MIT

- **TechStack**: Technologies used in the project
  - Add multiple technologies (JavaScript, Python, React, etc.)
  - Click to add from a predefined list
  - Must select at least one technology

#### Visual Content

- **Imágenes** (Images): Project screenshots
  - Upload multiple images showing your project
  - Drag and drop files into the upload zone
  - At least one image is required
  - Supported formats: JPG, PNG, GIF

<Note>
  All fields are required. The form will show validation errors if any field is incomplete or incorrectly formatted.
</Note>

## Managing Existing Projects

### Editing Project Information

<Steps>
  <Step title="Open Project">
    Navigate to "Proyectos" and click on the project you want to edit.
  </Step>
  
  <Step title="Enter Edit Mode">
    Click the "Editar" (Edit) button or navigate to the edit page from your projects list.
  </Step>
  
  <Step title="Update Information">
    The edit screen shows:
    - **Left Panel**: Form with all project details (name, summary, URLs, version, license, tech stack)
    - **Right Panel**: Project options and actions
  </Step>
  
  <Step title="Save Changes">
    After making changes to any field, click the save button to update the project.
  </Step>
</Steps>

## Project Publishing and Visibility

### Understanding Project Status

Projects can be in two states:

- **Publicado** (Published): Visible in the public projects section
- **Oculto** (Hidden): Only visible to you and collaborators

### Publishing or Hiding a Project

<Steps>
  <Step title="Go to Project Edit Page">
    Navigate to the project and click "Editar" (Edit).
  </Step>
  
  <Step title="Find Status Card">
    In the right sidebar, locate the "Estado del proyecto" (Project Status) card.
  </Step>
  
  <Step title="Check Current Status">
    The card displays:
    - Current status (Published in green or Hidden in red)
    - Description of what each status means
  </Step>
  
  <Step title="Toggle Status">
    Click the button:
    - "Publicar Proyecto" (Publish Project) - to make it visible
    - "Ocultar Proyecto" (Hide Project) - to make it private
  </Step>
</Steps>

<Tip>
  Keep projects hidden while still in development. Publish them when ready for public viewing.
</Tip>

## Managing Project Collaborators

### Adding Collaborators

<Steps>
  <Step title="Access Collaborators">
    From the project edit page, click "Gestionar" (Manage) in the "Gestionar Colaboradores" card.
  </Step>
  
  <Step title="Search for Users">
    Use the search bar to find students or other users by name or email.
  </Step>
  
  <Step title="Add Collaborator">
    Click on a user card to add them as a collaborator.
    - The system will show a confirmation
    - The user will appear in the collaborators list
  </Step>
  
  <Step title="View Current Collaborators">
    Click "Ver Colaboradores" (View Collaborators) to see all current project collaborators.
  </Step>
</Steps>

### Removing Collaborators

<Steps>
  <Step title="Open Collaborators Modal">
    Click "Ver Colaboradores" (View Collaborators).
  </Step>
  
  <Step title="Remove User">
    Find the collaborator you want to remove and click the remove button.
  </Step>
  
  <Step title="Confirm">
    The system will update immediately and the user will no longer be listed as a collaborator.
  </Step>
</Steps>

### Collaborator Display

On the public project page, collaborators are shown:
- With their initials in circular avatars
- Hovering over an avatar shows their full name and email
- Listed in the sidebar under "Colaboradores"

## Editing Project Content

### Using the Visual Content Editor

<Steps>
  <Step title="Access Content Editor">
    From the project edit page, click "Editar" in the "Editar Contenido" card.
  </Step>
  
  <Step title="Add Content Blocks">

  <Step title="Arrange Content">
    Drag and drop content blocks to reorder them.
  </Step>
  
  <Step title="Save Changes">

## Project Display Information

When users view your published project, they see:

### Main Content Area
- Project name as the main heading
- Summary text
- Image gallery (carousel or grid)
- Detailed content from the visual editor

### Sidebar Information
- **Author**: Your name and email with avatar
- **Colaboradores**: List of all collaborators with avatars
- **Versión**: Current project version
- **Licencia**: Project license
- **Enlaces**: Clickable links to repository and live demo
- **Tecnologías**: Tech stack as colored badges
- **Published date**: When the project was made public

## Version Management

### Updating Project Version

<Steps>
  <Step title="Edit Project">
    Go to the project edit page.
  </Step>
  
  <Step title="Update Version Field">
    Change the version number in the "Versión" field.
    - Follow semantic versioning: MAJOR.MINOR.PATCH
    - Example: 1.0.0 → 1.1.0 for new features
    - Example: 1.1.0 → 1.1.1 for bug fixes
  </Step>
  
  <Step title="Save">
    Save the changes. The new version appears on the public project page.
  </Step>
</Steps>

<Tip>
  Use version numbers to track project progress:
  - **1.0.0**: Initial release
  - **1.1.0**: New features added
  - **2.0.0**: Major changes or redesign
</Tip>

## Troubleshooting

### Cannot Save Project

**Possible causes**:
- Missing required fields
- Invalid URL format
- Version number doesn't match pattern (e.g., 1.0.0)

**Solution**: Check for red error messages under form fields and correct them.

### Collaborator Not Found

**Solution**: Ensure the user is registered in the system and has an active account. Use the search feature with different terms.

### Images Not Uploading

**Solution**: 
- Check file size (large files may fail)
- Ensure file format is supported (JPG, PNG, GIF)
- Try uploading one image at a time

### Project Not Visible After Publishing

**Solution**: Verify the project status shows "Publicado" in green. If it still doesn't appear, try logging out and back in.
