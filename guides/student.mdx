# Student Guide

Students in EduStack Smart can create their own projects, participate as collaborators, view events, and register for activities. This guide will help you make the most of your student account.

## Overview

As a student, you can:
- Create and manage your own projects
- Be added as a collaborator on teacher or peer projects
- View published blog posts
- Browse and register for events
- Update your project portfolios

## Working with Projects

### Creating Your Own Project

Students have full access to project creation features:

<Steps>
  <Step title="Navigate to Projects">
    Click on "Proyectos" (Projects) in the main navigation menu.
  </Step>
  
  <Step title="Start New Project">
    Click the "Crear Proyecto" (Create Project) button.
  </Step>
  
  <Step title="Complete Project Form">
    Fill in all required project information:
    - **Name**: Your project title
    - **Summary**: Description of what your project does (minimum 100 characters)
    - **Repository URL**: Link to your GitHub, GitLab, or other code repository
    - **Demo URL**: Link to your live project or demo
    - **Version**: Project version (e.g., 1.0.0)
    - **License**: Software license (default: MIT)
    - **Tech Stack**: Select technologies you used (JavaScript, Python, React, etc.)
  </Step>
  
  <Step title="Add Project Images">
    Upload at least one screenshot showing your project:
    - Drag and drop images into the upload zone
    - Multiple images can be added
    - Show different features or screens
  </Step>
  
  <Step title="Create Project">
    Click "Crear Proyecto" to save. Your project is created in hidden mode by default.
  </Step>
</Steps>

<Tip>
  Start with clear screenshots that showcase your project's main features. Good visuals make your project more engaging!
</Tip>

### Editing Your Projects

<Steps>
  <Step title="Open Your Project">
    Go to "Proyectos" and click on the project you want to edit.
  </Step>
  
  <Step title="Enter Edit Mode">
    Click the "Editar" (Edit) button.
  </Step>
  
  <Step title="Update Information">
    The edit page shows:
    - **Left**: Form with all project details you can modify
    - **Right**: Options for managing content, collaborators, and visibility
  </Step>
  
  <Step title="Save Changes">
    After updating any field, save your changes.
  </Step>
</Steps>

### Adding Detailed Content

Beyond the basic information, you can add rich content to your project:

<Steps>
  <Step title="Access Content Editor">
    From your project edit page, find the "Editar Contenido" card in the right sidebar.
  </Step>
  
  <Step title="Click Edit">
    Click "Editar" to open the visual content editor.
  </Step>
  
  <Step title="Build Your Content">
    Use the visual editor to:
    - Add formatted text with headings and paragraphs
    - Insert images and diagrams
    - Include code examples
    - Create custom layouts
    - Document features and how to use your project
  </Step>
  
  <Step title="Save and Preview">
    Save your content and view the project page to see how it looks.
  </Step>
</Steps>

## Managing Project Visibility

### Understanding Project Status

Your projects can be:

- **Oculto** (Hidden): Only you and your collaborators can see it
- **Publicado** (Published): Visible to everyone in the projects gallery

### Publishing Your Project

<Steps>
  <Step title="Go to Edit Page">
    Open your project and click "Editar".
  </Step>
  
  <Step title="Find Status Card">
    Look for "Estado del proyecto" (Project Status) in the right sidebar.
  </Step>
  
  <Step title="Check Status">
    The card shows your current status:
    - **Red text**: Project is hidden
    - **Green text**: Project is published
  </Step>
  
  <Step title="Publish">
    Click "Publicar Proyecto" (Publish Project) when your project is ready to share.
  </Step>
</Steps>

<Note>
  Keep your project hidden while you're still working on it. Publish only when it's ready for others to see!
</Note>

### When to Publish

Consider publishing when:
- All project information is complete and accurate
- Screenshots clearly show your project
- Repository and demo links are working
- Content documentation is finished
- You've tested everything thoroughly

## Collaborating on Projects

### Being Added as a Collaborator

Teachers or other students can add you as a collaborator:

1. **Notification**: You'll be added to a project by the project owner
2. **Access**: You can now view the project even if it's hidden
3. **Recognition**: Your name appears on the project page under "Colaboradores"

### Viewing Projects You Collaborate On

To see projects where you're a collaborator:

1. Navigate to the "Proyectos" section
2. Your collaborated projects appear in your project list
3. Your avatar shows on the project page alongside other collaborators

### Collaborator Display

On published projects, collaborators are shown:
- As circular avatars with initials
- With full name and email on hover
- Listed in the project sidebar

<Tip>
  Being a collaborator on projects is great for your portfolio! Make sure to add your contributions to collaborative projects.
</Tip>

## Updating Project Versions

As you improve your project, update the version number:

<Steps>
  <Step title="Edit Project">
    Open your project edit page.
  </Step>
  
  <Step title="Update Version Field">
    In the "Versión" field, change the version number:
    - **1.0.0**: Your first version
    - **1.1.0**: Added new features
    - **1.0.1**: Fixed bugs or small improvements
    - **2.0.0**: Major redesign or changes
  </Step>
  
  <Step title="Save">
    Save your changes. The new version appears on your project page.
  </Step>

## Viewing and Reading Blog Posts

### Accessing the Blog

<Steps>
  <Step title="Navigate to Blog">
    Click on "Blog" in the main navigation.
  </Step>
  
  <Step title="Browse Posts">
    View all published blog posts from members and teachers:
    - See post titles and summaries
    - View featured images
    - Check publication dates
    - Filter by categories
  </Step>
  
  <Step title="Read Full Post">
    Click on any post to read the complete content.
  </Step>
</Steps>

### Blog Post Features

When reading a post, you'll see:
- **Title**: Main heading
- **Author**: Who wrote the post
- **Summary**: Brief overview
- **Full Content**: Complete article with formatting, images, and media

- **Publication Date**: When it was published

## Exploring and Registering for Events

### Browsing Events

<Steps>
  <Step title="Navigate to Events">
    Click on "Eventos" (Events) in the main navigation.
  </Step>
  
  <Step title="View Event Listings">
    See all upcoming events with:
    - Event name and logo
    - Brief description
    - Dates and times
    - Location
    - Registration status
  </Step>
  
  <Step title="Open Event Details">
    Click on an event to see complete information.
  </Step>
</Steps>

### Event Information

Each event page shows:

- **Event Logo**: Circular image at the top
- **Title**: Event name
- **Organizer**: Name and contact of who created it
- **Summary**: Event description
- **Location**: Where the event takes place (with map if available)
- **Dates**: 
  - Registration period (when you can sign up)
  - Event start and end dates
- **Pricing**: 
  - Free or paid
  - Price amount (if not free)
  - Any discounts available
- **Detailed Content**: Full event information, schedule, speakers, etc.

### Registering for Events

<Steps>
  <Step title="Check Eligibility">
    Verify that:
    - Registration is currently open (between start and end dates)
    - You meet any requirements
    - You haven't already registered
  </Step>
  
  <Step title="Click Register">
    On the event page, click the registration button (if available).
  </Step>
  
  <Step title="Complete Registration">
    Follow the registration process:
    - Confirm your information
    - Pay if required (or confirm free registration)
    - Submit registration
  </Step>
  
  <Step title="Confirmation">
    You'll receive confirmation that you're registered for the event.
  </Step>
</Steps>

<Note>
  Registration deadlines are strictly enforced. Make sure to register before the "Fin de Registro" date!
</Note>

## Tips for Success

### Project Creation Tips

1. **Start Simple**: Begin with smaller projects and build up
2. **Use Real URLs**: Make sure repository and demo links actually work
3. **Write for Others**: Explain your project as if teaching someone new
4. **Show, Don't Tell**: Use screenshots to demonstrate features
5. **Version Consistently**: Follow semantic versioning (1.0.0, 1.1.0, etc.)

### Collaboration Tips

1. **Communication**: Coordinate with project owners about your role
2. **Credit Others**: Acknowledge when you're working with others
3. **Stay Updated**: Check projects you collaborate on regularly
4. **Contribute Meaningfully**: Add real value to collaborative projects

### Event Participation

1. **Register Early**: Don't wait until the last day of registration
2. **Read Details**: Fully review event information before registering
3. **Mark Calendars**: Add event dates to your calendar
4. **Prepare**: If it's a workshop or hands-on event, prepare in advance

## Troubleshooting

### Cannot Create Project

**Common issues**:
- Summary too short (needs 100+ characters)
- Invalid URL format
- No image uploaded
- Version number format incorrect

**Solution**: Look for red error messages under each field and fix them.

### Project Not Publishing

**Solution**: 
1. Make sure you clicked "Publicar Proyecto"
2. Check that status shows green "Publicado"
3. Refresh the projects page

### Cannot See Collaborated Project

**Solution**:
- Confirm you were added as a collaborator
- Check if the project still exists
- Contact the project owner to verify

### Event Registration Not Available

**Possible reasons**:
- Registration period hasn't started yet
- Registration deadline has passed
- Event is at capacity
- Event is no longer published

**Solution**: Check the registration start and end dates on the event page.
