# API Overview

The EduStack Smart API provides programmatic access to blog posts, events, projects, and user data. The API follows RESTful principles and returns JSON responses.

## Base URL

All API endpoints are relative to your application's base URL:

```
https://your-domain.com/api
```

For local development:

```
http://localhost/api
```

## API Prefix

All API routes are automatically prefixed with `/api`. For example:

- Blog endpoint: `/api/blog`
- User endpoint: `/api/user`

## Request Format

The API accepts standard HTTP methods:

- `GET` - Retrieve resources
- `POST` - Create new resources
- `PUT/PATCH` - Update existing resources
- `DELETE` - Remove resources

### Headers

For authenticated requests, include these headers:

```http
Content-Type: application/json
Accept: application/json
Authorization: Bearer {your-token}
```

## Response Format

All API responses are returned in JSON format with the following structure:

### Successful Response

```json
{
  "data": [
    {
      "id": 1,
      "name": "Resource name",
      // ... other fields
    }
  ],
  "links": {
    "first": "https://your-domain.com/api/blog?page=1",
    "last": "https://your-domain.com/api/blog?page=5",
    "prev": null,
    "next": "https://your-domain.com/api/blog?page=2"
  },
  "meta": {
    "current_page": 1,
    "from": 1,
    "last_page": 5,
    "path": "https://your-domain.com/api/blog",
    "per_page": 15,
    "to": 15,
    "total": 67
  }
}
```

### Paginated Collections

Collection endpoints return paginated results with metadata:

- `data` - Array of resources
- `links` - Navigation links (first, last, prev, next)
- `meta` - Pagination metadata (current_page, total, per_page, etc.)

## Error Handling

The API uses standard HTTP status codes to indicate success or failure:

### HTTP Status Codes

| Status Code | Description |
|------------|-------------|
| `200 OK` | Request succeeded |
| `201 Created` | Resource created successfully |
| `204 No Content` | Request succeeded with no response body |
| `400 Bad Request` | Invalid request parameters |
| `401 Unauthorized` | Authentication required or failed |
| `403 Forbidden` | Authenticated but not authorized |
| `404 Not Found` | Resource not found |
| `422 Unprocessable Entity` | Validation errors |
| `429 Too Many Requests` | Rate limit exceeded |
| `500 Internal Server Error` | Server error |

### Error Response Format

```json
{
  "message": "Error description",
  "errors": {
    "field_name": [
      "Validation error message"
    ]
  }
}
```

### Common Error Examples

**Validation Error (422)**

```json
{
  "message": "The given data was invalid.",
  "errors": {
    "email": [
      "The email field is required."
    ]
  }
}
```

**Unauthorized (401)**

```json
{
  "message": "Unauthenticated."
}
```

**Not Found (404)**

```json
{
  "message": "Resource not found."
}
```

## Rate Limiting

The API implements rate limiting to ensure fair usage and protect against abuse.

### Default Limits

Laravel's default API rate limiting applies:

- **60 requests per minute** per IP address for unauthenticated requests
- Rate limit information is included in response headers

### Rate Limit Headers

Each API response includes headers with rate limit information:

```http
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 59
Retry-After: 60
```

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed in the time window |
| `X-RateLimit-Remaining` | Requests remaining in current window |
| `Retry-After` | Seconds until rate limit resets (when limit exceeded) |

### Handling Rate Limits

When you exceed the rate limit, you'll receive a `429 Too Many Requests` response:

```json
{
  "message": "Too Many Attempts."
}
```

Your application should:

1. Monitor the `X-RateLimit-Remaining` header
2. Implement exponential backoff when approaching limits
3. Respect the `Retry-After` header when rate limited

## Pagination

Collection endpoints support pagination using query parameters:

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number to retrieve
</ParamField>

<ParamField query="per_page" type="integer" default="15">
  Number of items per page (Laravel default pagination)
</ParamField>

### Example Request

```bash
curl "https://your-domain.com/api/blog?page=2&per_page=10" \
  -H "Accept: application/json"
```

## Data Types

The API uses standard JSON data types:

- `string` - Text values
- `integer` - Whole numbers
- `float` - Decimal numbers
- `boolean` - `true` or `false`
- `array` - Ordered lists
- `object` - Key-value pairs
- `null` - Absence of a value

### Date Format

All dates and timestamps are returned in ISO 8601 format:

```
2026-03-04T10:30:00.000000Z
```

## API Resources

The API provides access to these resources:

- [Blog Posts](/api/blog) - Published blog posts with categories and authors
- [User Data](/api/users) - Authenticated user information
- [Projects](/api/projects) - Project listings (if available)
- [Events](/api/events) - Event listings (if available)

## Versioning

The current API does not use explicit versioning. Breaking changes will be communicated in advance.
