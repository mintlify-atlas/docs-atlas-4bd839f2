# Projects API

The Projects API would provide access to published projects with their authors, collaborators, and media.

## Current Status

<Warning>
There are currently **no public API endpoints** implemented for projects in `routes/api.php`.
</Warning>

However, the underlying infrastructure is in place to support project API endpoints:

## Available Resources

The codebase includes the following project-related resources:

### Models

**Project Model** (`app/Models/Project.php`)

The Project model includes these fields:

- `id` - Unique identifier
- `name` - Project name
- `slug` - URL-friendly identifier
- `summary` - Brief description
- `content` - Detailed content (JSON array)
- `repository_url` - Git repository URL
- `demo_url` - Live demo URL
- `tech_stack` - Technologies used (JSON array)
- `version` - Project version
- `license` - Software license
- `is_featured` - Featured status (boolean)
- `is_published` - Publication status (boolean)
- `published_at` - Publication timestamp
- `user_id` - Author/owner user ID
- `created_at` - Creation timestamp
- `updated_at` - Last update timestamp

### Relationships

```php
// Project belongs to a user (author)
public function author()
{
    return $this->belongsTo(User::class, 'user_id');
}

// Project has many collaborators (users)
public function collaborators()
{
    return $this->belongsToMany(User::class, 'project_collaborators', 
        'project_id', 'user_id')->withPivot('id');
}
```

### Media Collections

Projects support media using Spatie Media Library:

**Collection:** `screenshots` (stored on S3)

**Conversions:**
- `screenshot` - 1920x1080px, quality 85%, cropped, with responsive images

### API Resources

Pre-built API resource classes are available:

- `ProjectResource` - Transforms a single project
- `ProjectCollection` - Transforms a collection of projects

## Potential Implementation

To implement a projects API endpoint similar to the blog API, you would:

### 1. Create a Controller

```php
<?php

namespace App\Http\Controllers\API;

use App\Models\Project;
use Illuminate\Http\Request;
use App\Http\Controllers\Controller;
use App\Http\Resources\ProjectCollection;

class ApiProjectController extends Controller
{
    public function __invoke(Request $request)
    {
        $projects = Project::where('is_published', true)
            ->with(['author', 'collaborators'])
            ->orderBy('is_featured', 'DESC')
            ->paginate();

        return new ProjectCollection($projects);
    }
}
```

### 2. Add Route

In `routes/api.php`:

```php
Route::get('/projects', ApiProjectController::class)->name('api.projects');
```

### 3. Update ProjectResource

Ensure the `ProjectResource` includes media transformations similar to `PostResource`:

```php
public function toArray(Request $request): array
{
    $data = parent::toArray($request);

    $data['media'] = $this->getMedia('screenshots')->map(
        fn($m) =>
        new MediaResource($m, 'screenshot', [
            'screenshot' => ['width' => 1920, 'height' => 1080]
        ])
    );

    return $data;
}
```

## User Integration

Users can have relationships with projects:

### As Author

```php
// From User model
public function projects()
{
    return $this->hasMany(Project::class);
}
```

### As Collaborator

```php
// From User model
public function projectsCollaborations()
{
    return $this->belongsToMany(Project::class, 'project_collaborators', 
        'user_id', 'project_id')->withPivot('id');
}
```

## Related Models

### ProjectCollaborator

Pivot model for the many-to-many relationship between users and projects.
