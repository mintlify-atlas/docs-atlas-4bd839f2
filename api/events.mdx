# Events API

The Events API would provide access to published events with their details, locations, pricing, and media.

## Current Status

<Warning>
There are currently **no public API endpoints** implemented for events in `routes/api.php`.
</Warning>

However, the underlying infrastructure is in place to support event API endpoints:

## Available Resources

The codebase includes comprehensive event-related resources:

### Models

**Event Model** (`app/Models/Event.php`)

The Event model includes these fields:

- `id` - Unique identifier
- `name` - Event name/title
- `slug` - URL-friendly identifier
- `summary` - Brief description
- `content` - Detailed content (JSON array)
- `is_free` - Whether the event is free (boolean)
- `price` - Event price (float)
- `percent_off` - Discount percentage (float)
- `location` - Event location address
- `lat` - Latitude coordinate (float)
- `lng` - Longitude coordinate (float)
- `registration_started_at` - Registration start datetime
- `registration_ended_at` - Registration end datetime
- `start_date` - Event start date
- `end_date` - Event end date
- `is_published` - Publication status (boolean)
- `user_id` - Organizer/author user ID
- `event_status_id` - Current event status ID
- `created_at` - Creation timestamp
- `updated_at` - Last update timestamp

### Relationships

```php
// Event belongs to a user (organizer)
public function author()
{
    return $this->belongsTo(User::class, 'user_id');
}

// Event has many activities
public function activities()
{
    return $this->hasMany(EventActivity::class);
}

// Event belongs to a status
public function status()
{
    return $this->belongsTo(EventStatus::class, 'event_status_id');
}
```

### Media Collections

Events support media using Spatie Media Library:

**Collection:** `logo` (stored on S3, single file)

**Conversions:**
- `thumbnail` - 1080x1080px (square), quality 85%, cropped, with responsive images

### API Resources

Pre-built API resource classes are available:

- `EventResource` - Transforms a single event
- `EventCollection` - Transforms a collection of events

### Related Models

The event system includes extensive related models:

- `EventStatus` - Event status types (draft, upcoming, open, closed, ongoing, finished, canceled)
- `EventActivity` - Activities within events
- `EventActivityType` - Activity type categorization
- `EventActivityCategory` - Activity category classification
- `EventActivityRegistration` - User registrations for activities
- `EventActivityTeam` - Teams for team-based activities
- `EventActivityTeamMember` - Team member assignments
- `EventCollaborator` - Event collaborators/organizers
- `EventRegistration` - User registrations for events

## Potential Implementation

To implement an events API endpoint similar to the blog API, you would:

### 1. Create a Controller

```php
<?php

namespace App\Http\Controllers\API;

use App\Models\Event;
use Illuminate\Http\Request;
use App\Http\Controllers\Controller;
use App\Http\Resources\EventCollection;

class ApiEventController extends Controller
{
    public function __invoke(Request $request)
    {
        $events = Event::where('is_published', true)
            ->with(['author', 'status', 'activities'])
            ->orderBy('start_date', 'ASC')
            ->paginate();

        return new EventCollection($events);
    }
}
```

### 2. Add Route

In `routes/api.php`:

```php
Route::get('/events', ApiEventController::class)->name('api.events');
```

### 3. Update EventResource

Ensure the `EventResource` includes media transformations:

```php
public function toArray(Request $request): array
{
    $data = parent::toArray($request);

    $data['media'] = $this->getMedia('logo')->map(
        fn($m) =>
        new MediaResource($m, 'thumbnail', [
            'thumbnail' => ['width' => 1080, 'height' => 1080]
        ])
    );

    return $data;
}
```

## Event Status System

Events can have different statuses tracked through the `EventStatus` model:

- **Draft** - Event is being prepared
- **Upcoming** - Event is scheduled but registration hasn't opened
- **Open** - Registration is currently open
- **Closed** - Registration has closed but event hasn't started
- **Ongoing** - Event is currently happening
- **Finished** - Event has concluded
- **Canceled** - Event was canceled

The model includes commented logic for automatically calculating event status based on dates:

```php
// Automatically determine status based on:
// - Current datetime vs registration_started_at
// - Current datetime vs registration_ended_at  
// - Current datetime vs start_date
// - Current datetime vs end_date
```

## Event Activities

Events can have multiple activities (sessions, workshops, talks, competitions):

### EventActivity Model

Activities have their own:
- Types (`EventActivityType`)
- Categories (`EventActivityCategory`)
- Registrations (`EventActivityRegistration`)
- Teams (`EventActivityTeam`) with members

## Pricing Calculation

For paid events, the effective price can be calculated:

```javascript
// Calculate discounted price
const originalPrice = event.price;
const discount = event.percent_off;
const finalPrice = originalPrice * (1 - discount / 100);

// Example: $299.99 - 15% = $254.99
```

## Query Filtering Ideas

When implementing the API, consider adding filters:

- `status` - Filter by event status
- `is_free` - Show only free events
- `upcoming` - Show only future events
- `location` - Filter by location/region
- `date_from` / `date_to` - Date range filtering
- `price_max` - Maximum price filter
